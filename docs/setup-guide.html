1

このガイドでできること

このガイドに沿って作業すると、以下のようなツールが完成します:


  • 世界の業界ニュースを毎週自動で収集（英語ニュースを対象）
  • AIが日本語に翻訳・要約（3〜5行の簡潔なまとめ）
  • 見やすいWebページとして自動公開（HTMLファイル）
  • 毎週月曜朝8時にメールで通知（リンクが届く）

コードは一切自分で書く必要はありません。Claude Code（AI）に指示を出すだけで、全てのプログラムを自動生成してもらいます。

このガイドの例: 「塗装・塗装設備業界」のニュースを収集するツールを作成します。キーワードを変えれば、どの業界にも応用できます。

事前準備（Windows環境構築）

4つのソフトウェアをインストールします。既にインストール済みのものは飛ばしてください。

3-1. Node.js をインストール

必須

Claude Code の実行に必要です

  1. https://nodejs.org/ja にアクセス
  2. 左側の 「LTS（推奨版）」 をダウンロード
  3. ダウンロードしたインストーラーを実行し、全て「Next」で進む
  4. インストール完了後、PowerShell を開いて確認:
PowerShell node --version
npm --version

バージョン番号が表示されれば成功です。

3-2. Python をインストール

必須

ツール本体の実行に必要です

  1. https://www.python.org/downloads/ にアクセス
  2. 「Download Python 3.x.x」 をクリック
  3. インストーラーを実行
  4. 重要! 最初の画面で 「Add Python to PATH」にチェックを入れてから「Install Now」
  5. 完了後、PowerShell で確認:
PowerShell python --version

3-3. Git をインストール

必須

コードをGitHubにアップロードするために必要です

  1. https://git-scm.com/download/win にアクセス
  2. 「64-bit Git for Windows Setup」 をダウンロード
  3. インストーラーを実行し、全てデフォルト設定のまま「Next」で進む
  4. 完了後、PowerShell で確認:
PowerShell git --version

3-4. Claude Code をインストール

メインツール

AIがコードを書いてくれるツールです

  1. PowerShell を管理者として実行（スタートメニューで「PowerShell」を検索 → 右クリック → 「管理者として実行」）
  2. 以下のコマンドを実行:
PowerShell npm install -g @anthropic-ai/claude-code
  1. インストール完了後、確認:
PowerShell claude --version
  1. 初回起動して認証を行います:
PowerShell claude

ブラウザが開き、Anthropicアカウントでのログインを求められます。ログインすると Claude Code が使えるようになります。

ここまでの確認: PowerShell で node --version、python --version、git --version、claude --version がすべてバージョン番号を返せば準備完了です。

外部サービスのAPIキーを取得する

3つの外部サービスに登録し、APIキー（サービスを利用するための「鍵」）を取得します。

APIキーとは？
APIキーは、外部サービスにアクセスするための「パスワード」のようなものです。他人に教えないでください。このガイドでは、APIキーをGitHub Secretsという安全な場所に保管します。

4-1. NewsAPI キーの取得（無料）

世界のニュースを検索・取得するためのAPIです

  1. https://newsapi.org/ にアクセス
  2. 「Get API Key」 ボタンをクリック
  3. 名前・メールアドレス・パスワードを入力して登録
  4. ダッシュボードに表示される APIキー（英数字の文字列）をメモ帳にコピー
無料プラン: 月1,000リクエストまで。週1回の使用なら十分です。

4-2. Anthropic（Claude）API キーの取得

AIによる翻訳・要約に使います

  1. https://console.anthropic.com/ にアクセス
  2. Googleアカウント等でサインアップ/ログイン
  3. 左メニューの 「API Keys」 をクリック
  4. 「Create Key」 をクリック → 名前を入力（例: paint-news）
  5. 表示されたキー sk-ant-... をメモ帳にコピー
  6. 左メニューの 「Billing」 で支払い方法を登録
APIキーは一度しか表示されません！ 必ずその場でコピーしてメモ帳に保存してください。
費用目安: 週1回20記事の翻訳で月100〜200円程度です。

4-3. SendGrid API キーの取得（無料）

メール送信に使います

  1. https://sendgrid.com/ にアクセス
  2. 「Start For Free」 でアカウント作成
  3. ログイン後、左メニュー Settings → API Keys
  4. 「Create API Key」 をクリック
  5. 名前を入力、パーミッションは 「Full Access」 を選択
  6. 表示されたキー SG.xxxx... をメモ帳にコピー

送信元メールアドレスの認証（必須！）

  1. 左メニュー Settings → Sender Authentication
  2. 「Single Sender Verification」 をクリック
  3. 送信元にしたいメールアドレスを登録
  4. 確認メールが届くので、メール内のリンクをクリックして認証完了
ここまでの確認: メモ帳に以下の3つのキーが保存されていればOKです。
  • NewsAPIキー
  • Anthropic APIキー（sk-ant-...）
  • SendGrid APIキー（SG....）

Claude Code の基本操作

5-1. 起動と基本コマンド

起動方法

PowerShell で作業フォルダに移動してから claude と入力するだけです。

PowerShell # 1. 作業フォルダを作成して移動
mkdir C:\Projects\paint-news-app
cd C:\Projects\paint-news-app

# 2. Claude Code を起動
claude

起動すると入力プロンプトが表示され、自然な日本語で指示を出せます。

基本的な操作

操作方法
指示を出す日本語で入力して Enter
実行を許可するClaude が確認を求めたら y を入力
終了する/exit と入力、または Ctrl+C を2回
新しい会話を始める/clear と入力

5-2. 効果的なプロンプトの書き方

Claude Code に良い結果を出してもらうための 5つのコツ です:

コツ 1

最初に「全体像」を伝え、すぐに作業させない

いきなり「作って」ではなく、まず要件を整理してもらいましょう。

良いプロンプト例
業界ニュース自動まとめツールを作成して。 世界の塗装及び塗装設備業界のニュースを毎週自動で収集しHTMLファイルにまとめて、日本語に翻訳して毎週月曜の日本時間8時に、HTMLリンクをメールで通知するツールを作成してください。 まだ作業は行わず、まずは内容を確認してまとめてください、不明点は質問してください。
ポイント: 末尾の「まだ作業は行わず〜」が重要です。これにより Claude は要件を整理し、不明点を質問してくれます。認識のズレを防げます。
コツ 2

具体的な数字・技術・条件を含める

曖昧な表現を避け、具体的に指定するほど良い結果が得られます。

曖昧な指示具体的な指示
ニュースを集めてNewsAPIを使って過去7日間のニュースを最大20件収集して
日本語にしてClaude APIで日本語に翻訳し、3〜5行で要約して
定期的に実行してGitHub Actionsで毎週月曜の日本時間8:00に自動実行して
きれいに表示してシンプル・ミニマルなデザインのHTMLファイルにまとめて
コツ 3

品質を重視する場合は明示する

品質やこだわりポイントは、はっきりと伝えましょう。

効果的な一文
時間はいくらかけても良いので品質を優先してください。

この一文を添えると、Claude はより丁寧にコードを書き、エラーハンドリングやコメントも充実させます。

コツ 4

段階的に進める（一度に全部頼まない）

大きなタスクは段階的に分けると、品質が上がります。

段階指示の例
1. 要件整理「まず要件をまとめて、不明点を質問して」
2. 作成指示「この内容で作業を進めてください」
3. テスト「ドライランで動作確認してください」
4. 修正「メール送信でエラーが出ます。ログはこちらです: ...」
コツ 5

エラーが出たら「ログをそのまま貼る」

問題が起きた時は、自分で解釈せず、エラーメッセージやログをそのままコピー&ペーストしてください。

エラー報告の例
メールが届きません。GitHubのActionsのログは以下の通りです: SendGrid API エラー: HTTP Error 401: Unauthorized
ポイント: エラーの原因は Claude が判断します。あなたはログを貼るだけでOKです。

ツールを作成する（プロンプト全文）

いよいよツールを作成します。以下のプロンプトを 順番に Claude Code に送信してください。

プロンプト 1 / 4

Claude Code を起動して、要件を伝える

まず PowerShell で作業フォルダに移動し、Claude Code を起動します:

PowerShell mkdir C:\Projects\paint-news-app
cd C:\Projects\paint-news-app
claude

起動したら、以下のプロンプトを送信:

プロンプト 1 をコピー
業界ニュース自動まとめツールを作成して。 世界の塗装及び塗装設備業界のニュースを毎週自動で収集しHTMLファイルにまとめて、日本語に翻訳して毎週月曜の日本時間8時に、HTMLリンクをメールで通知するツールを作成してください。 技術スタック: - ニュース収集: NewsAPI - 翻訳・要約: Claude API - 定期実行・ホスティング: GitHub Actions + GitHub Pages - メール送信: SendGrid - 言語: Python まだ作業は行わず、まずは内容を確認してまとめてください。不明点は質問してください。

Claude が質問してくるので、以下のように回答してください:

  • ニュース収集キーワード → 業界全般（幅広く収集）
  • 記事数 → 20件程度
  • デザイン → シンプル・ミニマル
  • 要約レベル → 3〜5行の標準要約
プロンプト 2 / 4

作成を指示する

Claude が要件をまとめてくれたら、作成を指示します:

プロンプト 2 をコピー
はい、この内容で作業を進めてください。時間はいくらかけても良いので品質を優先してください。

Claude がファイルを作成し始めます。途中で「許可しますか？」と聞かれたら y を入力してください。

所要時間: 10〜20分ほどかかります。Claude が全てのファイルを自動で作成します。
プロンプト 3 / 4

ドライランでテストする

ファイル作成が完了したら、テスト実行を依頼します:

プロンプト 3 をコピー
ドライランで動作確認してください。APIは使わず、ダミーデータでHTMLが正しく生成されるか確認したいです。
プロンプト 4 / 4

セットアップガイドを聞く

テストが成功したら、デプロイ手順を聞きます:

プロンプト 4 をコピー
運用開始手順を詳しく教えてください。GitHubリポジトリの作成からGitHub Pages有効化、GitHub Secretsの設定、テスト実行まで、Gitに不慣れなので全てのコマンドを省略せず教えてください。
業界を変えたい場合は？
プロンプト1の「塗装及び塗装設備業界」を「半導体業界」「食品加工業界」など、お好みの業界に置き換えるだけです。Claude が適切なキーワードを自動で設定してくれます。
7

GitHubにデプロイする

Claude Code が手順を教えてくれますが、ここでも流れをまとめておきます。

7-1. GitHubアカウントの準備

アカウントをお持ちでない場合は github.com で無料登録してください。

7-2. リポジトリの作成とプッシュ

  1. https://github.com/new にアクセス
  2. Repository name: paint-news-app
  3. 「Public」 を選択（GitHub Pages無料利用のため）
  4. 「Add a README file」にはチェックしない
  5. 「Create repository」をクリック

次に PowerShell で以下を実行:

PowerShell cd C:\Projects\paint-news-app

git init
git add .
git commit -m "初回コミット"
git branch -M main

# あなたのユーザー名に書き換えてください
git remote add origin https://github.com/あなたのユーザー名/paint-news-app.git

git push -u origin main

7-3. GitHub Secrets の登録

GitHubリポジトリ → Settings → Secrets and variables → Actions → 「New repository secret」 で以下を登録:

Name登録する値
NEWSAPI_KEYNewsAPIで取得したキー
ANTHROPIC_API_KEYsk-ant-... のキー
SENDGRID_API_KEYSG.... のキー
FROM_EMAILSendGridで認証した送信元メールアドレス
NOTIFY_EMAIL通知を受け取りたいメールアドレス
PAGES_BASE_URLhttps://あなたのユーザー名.github.io/paint-news-app/
注意: Name の「GITHUB_」で始まる名前は使えません（GitHubの予約語）。PAGES_BASE_URL を使ってください。

7-4. GitHub Pages の有効化

  1. リポジトリの Settings → Pages
  2. Branch: main を選択
  3. フォルダ: /docs を選択
  4. 「Save」 をクリック
リポジトリが Private になっている場合は、Settings 最下部の「Change visibility」から Public に変更してください。
8

運用開始・メンテナンス

8-1. テスト実行

  1. リポジトリの 「Actions」 タブをクリック
  2. 左側の 「Weekly Paint Industry News」 をクリック
  3. 「Run workflow」 ボタンをクリック
  4. ドライラン: true、メール送信スキップ: true で最初のテスト
  5. 成功したら、両方 false にして本番テスト

8-2. 自動配信の確認

設定完了後は、毎週月曜 朝8時（日本時間） に自動で以下が実行されます:

  1. 過去7日間のニュースを自動収集
  2. AIが日本語に翻訳・要約
  3. HTMLレポートを自動生成・公開
  4. メールでリンクを通知

あなたがやることは何もありません。毎週メールを確認するだけです。

8-3. カスタマイズしたいとき

設定を変えたい場合は、再度 Claude Code を起動して指示を出すだけです:

カスタマイズ例
検索キーワードを変更して、「半導体製造装置」のニュースも含めるようにしてください。
カスタマイズ例
記事の最大件数を30件に増やして、配信を水曜と月曜の週2回に変更してください。

変更後は git add . → git commit -m "設定変更" → git push でGitHubに反映します。

9

トラブルシューティング

GitHub Actions が赤い ✗ で失敗する

Actions タブでエラーのあるジョブをクリックし、赤い行のログを確認してください。多くの場合、Secretsの設定ミスが原因です。

  • Secret の Name が正しいか確認（大文字・アンダースコア注意）
  • Secret の値に余分なスペースが入っていないか確認
メールが届かない
  • 迷惑メールフォルダを確認
  • Actions ログの Step 4/4 を確認し、エラーメッセージを確認
  • 401 Unauthorized → SendGridのAPIキーを再生成して登録し直す
  • SendGridの送信元メール認証が完了しているか確認
ニュースが0件になる
  • ニッチすぎるキーワードの場合、該当記事が無い週があります
  • Claude Code で検索キーワードを追加してもらいましょう
  • NewsAPIの無料枠（月1,000回）を超えていないか確認
GitHub Pages にアクセスできない
  • リポジトリが Public になっているか確認
  • Settings → Pages で Branch が main、フォルダが /docs に設定されているか確認
  • デプロイ後、反映まで数分かかることがあります
Claude Code でエラーが出る
  • Node.js が正しくインストールされているか node --version で確認
  • npm install -g @anthropic-ai/claude-code でインストールし直す
  • Anthropicアカウントのログインが切れている場合は再度 claude で起動してログイン
どんな問題でも、Claude Code に聞けば解決策を教えてもらえます。
エラーメッセージやスクリーンショットをそのまま貼り付けて「これを解決して」と伝えるだけでOKです。
